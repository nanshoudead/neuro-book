# Subject 文件架构

本文档详细说明 `simulation/subjects/{subject-id}/` 目录的六个文件职责、信息分流规则与 RAG 机制。

## 文件职责总览

每个 subject 目录包含六个文件，分工如下：

| 文件 | 稳定性 | 可见性 | 职责 |
|------|--------|--------|------|
| `soul.md` | 跨剧情恒定 | Import 进 actor 主路 | 第一人称扮演手册：性格、说话方式、角色恒定常识 |
| `subject.md` | 跨剧情恒定 | 仅 simulator.leader | 全知秘密档：隐藏真相、作者意图、角色不知道的设定 |
| `events.jsonl` | 剧情累积 | RAG 索引 | 角色第一人称经历流：发生了什么、我当时怎么想/误解/推理 |
| `memory.jsonl` | 剧情累积 | RAG 索引 | 角色对特定主体的稳定看法：我对 XX 的态度/关系/误解 |
| `mind.md` | 易变 | actor.context-load 读 | 当前想法：此刻在想什么、疑虑、短期意图 |
| `state.md` | 易变 | 上级读取 | 可见状态：位置、装备、伤势、外观变化 |

## 信息分流铁律

### soul.md 只写跨剧情恒定的

**核心原则**：soul.md 是角色的"扮演手册"，写的是这个角色在任何剧情中都成立的信息。

✅ **正确示例**：
- "我是布劳尔子爵领的世袭领主，这个头衔传了七代"
- "我是这座城堡的女仆长，在这里工作了十年"
- "我的性格底色是谨慎、不轻信新人"

❌ **错误示例**（这些是剧情，应写在 events.jsonl）：
- "我刚举行了召唤仪式，来了四个人"
- "我被召唤到中世纪城堡大厅，同时被召唤的还有三个人"
- "现在他们来了。四个人。三个展现了耀眼的能力"

**判断标准**：如果这句话在下一个剧情中可能不成立（例如"四个人"的数量、"刚举行"的时态），那它就不该出现在 soul.md。

### "我知道什么" vs memory.jsonl

这是最容易混淆的两个信息出口。区分原则：

**soul.md "我知道什么"**：角色职业/身份层面本来就知道的常识
- ✅ "我知道宅邸布局、仆人调度、基本礼仪规矩"
- ✅ "我知道基本的魔法常识（贵族家会用召唤术式），但我自己不会施法"
- ✅ "我知道子爵家族已经衰败，但我不完全清楚财务细节"

**memory.jsonl**：对特定人/事的动态看法，可能随剧情变化
- ✅ `{"topic":"当前主人（布劳尔子爵）","view":"比他父亲心软，但最近越来越焦虑..."}`
- ✅ `{"topic":"召唤仪式结果","view":"四名异界者到来，三个展现了能力，一个还没表现..."}`

**反例对比**（基于实际问题案例）：

❌ **错误**：同样的信息在两处重复
- soul.md "我知道什么" 写："我知道召唤仪式成功了，四名异界者来了，领地面临困境"
- memory.jsonl 写：`{"topic":"召唤仪式","view":"召唤成功，四个人来了，领地有困境"}`

✅ **正确**：恒定常识 vs 动态认知
- soul.md "我知道什么" 写："我知道召唤术式的基本原理（贵族常用手段），但我不会施法"
- memory.jsonl 写：`{"topic":"召唤仪式结果","view":"四名异界者到来，三个展现了能力（火焰、冰霜、光芒），一个还没表现。我需要观察他们的忠诚度和实际战力。"}`

### events.jsonl vs memory.jsonl 的区别

| 维度 | events.jsonl | memory.jsonl |
|------|-------------|-------------|
| 内容性质 | 时间轴事实流，第一人称经历 | 对某主体的当前认知快照 |
| topic 字段 | 无 topic（按时间索引） | topic 是主体名（人/事/势力） |
| 时间性 | 固定时间点，历史事实 | 当前认知，可能随剧情更新 |
| 累积方式 | 只追加，不修改历史 | 可覆盖旧认知 |
| 典型内容 | "我经历了什么""我当时怎么想/推理/误解" | "我对 XX 的看法""我对 XX 的关系定位" |

**示例对比**：

**events.jsonl**（历史经历）：
```jsonl
{"text":"旧子爵去世那年，府里裁了一半仆人。我因为手脚利索、嘴严被留下，升为管家。从那以后我知道：贵族家的体面是用仆人的勤快撑起来的。","time":"复兴纪元470年"}
{"text":"最近主人越来越焦虑，长袍破了都不舍得换。我听说他在密室里捣鼓什么仪式——但那不是我该问的。","time":"复兴纪元488年 风信之月"}
```

**memory.jsonl**（当前认知）：
```jsonl
{"topic":"当前主人（布劳尔子爵）","view":"比他父亲心软，但最近越来越焦虑。我看着他从意气风发的年轻领主变成现在这副憔悴样子——这让我想起旧子爵去世前的那段日子。"}
```

## RAG 机制

### 索引源与检索逻辑

actor.context-load 通过 `subject_rag_search` 工具从 events.jsonl / memory.jsonl 检索相关记忆注入主路。

**索引源**：只有 events.jsonl 和 memory.jsonl 进入 RAG 索引。

**不进索引的文件**：
- soul.md：永不进 RAG 索引（始终全文 Import，是信息控制最后一道闸）
- subject.md：永不进 RAG 索引（仅 leader 可见，actor 不可知）
- mind.md / state.md：不进索引（actor.context-load 直接读全文）

### 初始化种子的重要性

**问题**：如果初始化时 events.jsonl 和 memory.jsonl 是空的，会发生什么？

**答案**：RAG 检索不到任何东西——actor 只有 soul.md 撑场。

**后果**：
- 角色技能/知识无法溯源（例如"我是资深管家"但没有 events 记录学徒经历）
- 角色对其他主体的初始态度无法体现（例如"我对旧主人的怀念"只在 soul 提了一句，但没有具体 memory）
- RAG 空转，actor 可能编造不存在的记忆

**解决方案**：初始化时必须写"人生履历种子"（events）和"稳定认知种子"（memory）。详见 [subject-creation-guide.md](../agent/rp-tick/subject-creation-guide.md) Step 4/5。

## 常见错误与修正

### 错误 1：soul.md "我是谁"混入剧情

**实际案例**（来自 workspace/ming-ding-zhi-shi-2/simulation/subjects/armand-brauer）：

```markdown
# 我是谁
...
现在他们来了。四个人。三个展现了耀眼的能力——火焰、冰霜与光芒。
还有一个女人，她还没展示什么，但可能比前三个更危险。
```

**问题**：
- "现在他们来了"：时态绑定当前剧情，不跨剧情恒定
- "四个人"：数量可能变化（如果下次剧情召唤五个人，这句话就错了）
- 这些是 Tick 000001 的具体剧情，应写在 events.jsonl

**正确修正**：

soul.md "我是谁"：
```markdown
我是布劳尔子爵领的世袭领主，这个头衔传了七代。
领地在我手上衰败了——产出不足、税收锐减、几乎破产。
我知道传统手段救不了这里，所以我启用了禁忌手段：召唤异界者。
```

events.jsonl：
```jsonl
{"text":"我完成了召唤仪式。四个人出现在大厅——三个瞬间展现了能力（火焰、冰霜、光芒），一个女人还没表现，但我的直觉告诉我她可能更危险。仪式成功了，但接下来怎么用好他们才是关键。","time":"复兴纪元488年 风信之月 新月之夜"}
```

### 错误 2：soul.md "我知道什么" 与 memory.jsonl 重复

**实际案例**（来自同一角色）：

soul.md "我知道什么" L30-38：
```markdown
- 我知道召唤仪式成功了
- 我知道四名异界者到来
- 我知道领地面临困境
```

memory.jsonl L1-4：
```jsonl
{"topic":"召唤仪式结果","view":"召唤成功，四个异界者来了，领地有困境..."}
```

**问题**：同样的信息用不同措辞在两处出现，未区分"角色恒定常识" vs "对特定主体的动态看法"。

**正确分流**：

soul.md "我知道什么"：
```markdown
- 我知道召唤术式的基本原理（贵族常用手段），需要祭品、魔阵、咒文
- 我知道异界者通常比本土人更强，但忠诚度难以保证
- 我知道领地的产业结构（农业为主，手工业衰退）和财务状况（几乎破产）
```

memory.jsonl：
```jsonl
{"topic":"召唤仪式结果","view":"四名异界者到来，三个展现了能力（火焰、冰霜、光芒），一个还没表现。我需要观察他们的忠诚度和实际战力，然后决定派给他们什么任务。"}
{"topic":"领地困境","view":"产出不足、税收锐减、几乎破产。近郊农田受虫害，佃农流失。我试过传统手段（减税、招商）都没用，所以才启用召唤术。"}
```

### 错误 3：初始化时 events/memory 是空的

**问题**：
- 角色 soul.md 写"我是资深管家，在家族服务了三十年"
- 但 events.jsonl 是空的，没有任何"我如何成为管家""我经历过什么家族变故"的记录
- actor 回答"你为什么这么忠诚"时只能泛泛说"职业习惯"，无法溯源到具体经历

**解决方案**：

初始化时写 3-10 条人生履历种子（events.jsonl）：
```jsonl
{"text":"我十五岁进入布劳尔家族当学徒女仆，老管家教我'伺候贵族第一条：嘴严、手快、眼力见儿好'。这句话我记了三十年。","time":"三十年前"}
{"text":"旧子爵去世那年，府里裁了一半仆人。我因为手脚利索、嘴严被留下，升为管家。从那以后我知道：贵族家的体面是用仆人的勤快撑起来的。","time":"复兴纪元470年"}
```

初始化时写 1-3 条稳定认知种子（memory.jsonl）：
```jsonl
{"topic":"当前主人（布劳尔子爵）","view":"比他父亲心软，但最近越来越焦虑。我看着他从意气风发的年轻领主变成现在这副憔悴样子。"}
{"topic":"仆人职责","view":"老管家教我的第一条：嘴严、手快、眼力见儿好。贵族家的体面是用仆人的勤快撑起来的。我不该问的不问，不该说的不说——但我会把该做的事做到位。"}
```
