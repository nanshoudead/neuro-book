# Actor-Facing Packet 标签规范

actor-facing packet 是 simulator.leader 发给 simulator.actor 的消息。核心约束：**只包含该角色在戏内合理能感知、知道或推断的信息**。

## 标签总览

| 标签 | 用途 | 必须 |
|------|------|------|
| `<gm>` | 场景描述 + 环境事件 | 是 |
| `<character name="...">` | 其他角色的可观察行为和台词 | 有其他角色行为时 |
| `<knowledge>` | 该角色合理已知的世界知识 | 需要补充角色常识/专业知识时 |
| `<directive>` | 本轮特殊指示 | 可选 |

---

## `<gm>`

场景描述 + 环境事件。相当于 GM 给角色描述"你看到了什么、听到了什么、感受到了什么"。

**内容来源**：
- 当前场景环境（从 `current.md`）
- LOD0 中该角色能感知到的事件
- LOD1 中能传到当前场景的感知（远处的声音、气味等）

**规则**：
- 用第二人称"你"描述
- 只写该角色能直接感知的（视觉、听觉、触觉、嗅觉）
- 不写其他角色的内心
- 不写该角色不知道的设定原理
- LOD 事件中的 lorebook 术语必须转换为该角色认知水平的描述

**示例**：

```xml
<gm>
你站在仪式大厅台阶之上，子爵正在你旁边笨拙地回应那个金色光幕少年的质问。

一阵风从西侧没关紧的窗户灌进来，旁边三根蜡烛同时晃了一下，
蜡油落在铁烛台上啪嗒响了几声。横梁上一只灰鸽子被下方越来越大的声音惊到，
扑棱着飞到另一根横梁上，带下来几片灰尘。

你注意到那个没有任何外放的白发小姑娘动了。
她悄悄往眼镜女孩的方向移动了几步，然后小声说了什么。
</gm>
```

## `<character name="...">`

其他角色的可观察行为和台词。"你看到/听到某某做了什么、说了什么"。

**规则**：
- `name` 填角色在该 actor 认知中的称呼
  - 认识的人用名字
  - 不认识的人用描述性称呼："白发女孩""穿红袍的老人"
- 只写可观察的外在行为：动作、表情、台词、声音
- 不写该角色的内心想法

**示例**：

```xml
<character name="薇洛丝">
那个一直安静站着的白发女孩悄悄走到了你旁边。
她侧过头，小声问你："有没有事？"
</character>

<character name="运动男生">
他的金色光幕更亮了。他握紧拳头朝前迈了半步。
"保证？凭什么？你把我们莫名其妙弄到这里，连到底发生了什么都不说，
现在就要我们'随你走'？"
</character>
```

## `<knowledge>`

注入该角色合理已知的世界知识、背景信息、专业判断依据。

**解决的问题**：actor 的 sidecar 只加载了 memory/events/mind，但角色还有**常识性知识**或**专业知识**不在这些文件里。

**规则**：
- 只注入该角色合理知道或能推断的知识
- 用角色自己的认知水平描述，不用超越其知识边界的术语
- 不注入该角色不应知道的隐藏设定

**示例——法师（魔法专家）**：

```xml
<knowledge>
你是受过正规训练的法师。你知道：
- 异界召唤术式成功时，被召唤者通常会立即展现某种形式的魔力外放——光幕、元素亲和、感知强化等。
- 完全没有外放的情况极为罕见，通常意味着召唤失败（灵魂碎片丢失）或者能力处于某种封印/休眠状态。
- 后者比前者罕见得多，但并非不可能。历史上有少数案例。
- 白发女孩虽然没有外放，但也没有召唤失败常见的不适症状——眩晕、呕吐、意识模糊。这很异常。
</knowledge>
```

**示例——子爵（领地主人）**：

```xml
<knowledge>
你清楚领地目前的困境：
- 连续两季税收不达标，金库接近枯竭。
- 近郊农田近期出现了不明虫害，已经毁了三块麦田，进一步威胁粮食安全。
- 你举行召唤仪式的真正原因不仅是"古老预言"，更是绝望之举——领地急需异界者的力量。
- 你不能在卫兵面前暴露这些，否则军心会动摇。
</knowledge>
```

**示例——普通高中生**：

```xml
<knowledge>
你是一个普通的高中生。你不理解这个世界的任何规则。
你脚下的发光文字是什么、为什么你的手指在微微发麻，你完全不知道。
你只知道：你和另外三个人被传送到了一个陌生的地方，
一个穿红袍的老人说你们被"召唤"来了。
</knowledge>
```

## `<directive>`（可选）

simulator.leader 给 actor 的本轮特殊指示。不是每轮都需要。用于微调角色行为、引导剧情方向。

**适用场景**：
- Plot 需要角色做出某个特定反应
- 角色处于特殊状态需要提醒（受伤、醉酒、魔法影响等）
- simulator.leader 想引导但不强制某个方向

**规则**：
- 用建议而不是命令（"你可能会……"而不是"你必须……"）
- 不破坏角色的自主性——actor 可以根据角色性格偏离指示
- 不用来注入该角色不知道的信息

**示例**：

```xml
<directive>
你已经被这个年轻人质问了快三分钟，你的耐心在消耗。
你开始考虑要不要让法师出面，或者直接命令卫兵把他们带走。
但你也意识到动用武力可能适得其反——这些人有超自然能力。
</directive>
```

## 完整 packet 示例

以下是一个发给 glasses-girl 的完整 packet：

```xml
<knowledge>
你是一个普通的高中生。你不理解这个世界的任何规则。
你脚下的发光文字是什么、为什么你的手指在微微发麻，你完全不知道。
</knowledge>

<gm>
你正站在仪式大厅里，紧紧抱着帆布背包。脚下有一圈淡蓝色的发光文字在缓缓转动。

一阵风从旁边没关紧的窗户灌进来，几根蜡烛同时晃了一下，
蜡油在铁烛台上啪嗒响了几声。横梁上一只灰鸽子被下方的声音惊飞了，
带下来几片灰尘。

你突然注意到脚下的发光文字好像变色了——原本是淡蓝色的，
但一束从彩色玻璃窗投过来的红色光斑正好落在上面，
现在泛出了一层淡淡的紫色。你不知道那意味着什么。
</gm>

<character name="运动男生">
他的金色光幕更亮了。"保证？凭什么？你把我们莫名其妙弄到这里，
连到底发生了什么都不说，现在就要我们'随你走'？"
他朝台阶上的老人迈了半步。卫兵们把长戟往前推了推。
</character>

<character name="薇洛丝">
那个一直安静站着的白发女孩——她身上没有任何发光或能力外放——
悄悄走到了你旁边。她侧过头，小声问你："有没有事？"
</character>
```

## simulator.actor report_result

actor 的 report_result 使用**第一人称**描述。三通道：

| 字段 | 含义 |
|------|------|
| `visible_response` | 其他人能看见的（动作、神态、沉默） |
| `spoken_dialogue` | 说出口的话 |
| `inner_response` | 没说出口的（情绪、意图、误解） |

```json
{
  "visible_response": "我微微一颤，下意识把背包抱得更紧了。然后慢慢转过头，眨了眨眼睛——真的有人在跟我说话吗？",
  "spoken_dialogue": "我……我没事。你、你也是被召唤过来的吧？你……你不害怕吗？",
  "inner_response": "终于有人搭话了。这个白发女孩看起来好冷静，明明她也什么能力都没有……她怎么做到这么平静的？"
}
```
