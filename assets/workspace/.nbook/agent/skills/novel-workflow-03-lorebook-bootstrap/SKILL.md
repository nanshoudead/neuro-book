# novel-workflow-03-lorebook-bootstrap：世界书初始化

在已有小说基础定位上，协助用户初步完善 lorebook。默认目标是建立开篇可用的基础条目，不是一次性写完整百科全集，也不是剧情规划；它只把会影响后续剧情、角色行动和写作判断的世界真相写清楚。

如果用户明确想继续扩展世界书，可以在基础条目完成后继续深入历史、势力、地点、力量体系、普通生活、资源生态等模块。扩展时仍按用户意图和当前作品需要推进，不为了完整性机械补全百科。

## lorebook 与 World Engine 各管什么

lorebook 不是世界状态的唯一去处。它和 World Engine（世界引擎）职责互补：

- **lorebook 存静态设定**：世界观、规则、身份底设、力量体系、地理、势力关系——这些是不随剧情时间改变的 canon。例如「艾莉娜是凤凰骑士团成员」「灵气枯竭会引发狂化」。
- **World Engine 存动态状态与时间线**：随剧情演变的东西——谁现在在哪、HP 多少、当前关系如何、何时发生了什么。例如「艾莉娜在某年加入骑士团」「她现在 HP 80/100，身处星陨遗迹」。

判断一条信息归谁，只问一句：**它会不会随剧情时间演变？** 会演变的进 World Engine，稳定不变的 canon 进 lorebook。本流程只负责写 lorebook 这一侧的静态设定；动态状态留到 World Engine 初始化时录入。原理见 `reference/world-engine/workflow.md` 第 2 节。

lorebook 设定可以在初始化 World Engine 时映射成结构化属性：lorebook 里写的力量体系等级、货币、阵营、状态规则，到 `novel-workflow-world-engine-init` 时可映射成 character 等 subject 的 schema 属性。一句话区分：**lorebook 定义「是什么」，World Engine 记录「何时变成什么」**。所以这里把设定写清楚、写稳定即可，不必担心当前状态没处放。

## 前置要求

开始前应至少具备：

- 基础题材与基调。
- 核心 hook 或读者承诺。
- 主角基础定位。
- 简介或粗略剧情种子。

如果这些基础仍然缺失，先回到 `novel-workflow-02-project-bootstrap`。

## 工作方式

本流程应先读取已有初始化节点和相关 lorebook，再判断缺口。需要连续推进时，用 task_create 建立 task checklist，并用 task_set_status 一步步推进；不要一次性写完再汇报。

建议 task checklist：

1. 读取小说基础定位、简介和已有 lorebook。
2. 用一个异常启动世界观。
3. 从异常推导规则、资源、阶层、制度和矛盾。
4. 从矛盾推导历史、势力和普通生活。
5. 补齐主角设定、关键人物、地点与当前局势所需条目。
6. 写入或更新 lorebook 内容节点和必要 state.md。
7. 汇报世界书初始化结果、待定问题和下一步建议。

如果用户只要求补某一块，只做目标范围；但仍要检查它和世界规则、资源、势力或日常生活是否自洽。

## 推导链

优先用下面这条链条建立世界逻辑。不是每个世界都需要完整展开；不适合的环节可以简化、弱化或省略，但不要跳过因果解释。

每一步都不要只给一个直觉答案。先从不同角度做多次推导，再比较后收束：

- 物理角度：规则、能量、生态、资源、技术或力量如何运转。
- 社会角度：谁获利、谁受损、谁解释规则、谁维持秩序。
- 日常角度：普通人的工作、教育、交易、信仰、风险和语言习惯如何变化。
- 剧情角度：这些设定怎样给主角、关键人物和开局地点制造压力。

通常先提出 2 到 3 个可行解释或后果，再选择最能服务题材、主角处境和核心矛盾的版本写入 lorebook。用户只要求局部补充时，也要做小范围多角度检查。

### 1. 用一个异常启动世界观

先回答：这个世界和现实世界最大的不同是什么？

异常可以是：

- 物理异常：灵气、魔法、超能力、异星生态、时间循环、神明显现。
- 社会异常：身份制度、资源分配、记忆交易、阶层锁死、全民契约。
- 历史异常：灾变后秩序、某场战争的长期后果、旧文明遗产。
- 认知异常：人类不知道某个真相，或世界运转依赖错误信念。

### 2. 从异常推出规则

把异常变成可解释的基本规则：

- 它从哪里来。
- 它如何运作。
- 它有什么代价、限制或边界。
- 它是否会衰减、枯竭、污染、失控或进化。
- 普通人如何理解它，专家或统治者又如何理解它。

### 3. 从规则推出资源

继续问：在这些规则下，什么东西会变得稀缺、有价值或危险？

资源可以是能量、材料、知识、血脉、地点、资格、时间、安全区、身份、信仰或信息。

### 4. 从资源推出阶层

资源会塑造人群差异：

- 谁能获得资源。
- 谁被排除在资源之外。
- 谁负责生产、运输、守护、分配或解释资源。
- 阶层是否稳定，普通人是否有上升通道。

乌托邦、荒野求生、孤岛或小共同体题材可以弱化阶层，但仍要说明资源如何影响人的处境。

### 5. 从阶层推出制度

制度是资源和阶层长期稳定后的结果：

- 国家、城市、宗门、公司、学院、部落、家族或教会如何管理资源。
- 法律、禁令、考试、军队、税赋、交易、婚姻或身份制度如何运作。
- 制度维护了谁的利益，又牺牲了谁。

### 6. 从制度推出矛盾

矛盾要能给剧情施压：

- 被压迫者想改变什么。
- 既得利益者害怕失去什么。
- 制度内部有什么漏洞、腐败、误判或不可持续之处。
- 主角为什么会被卷入，或为什么只有主角能看见问题。

### 7. 从矛盾推出历史

历史不是年表堆砌，而是解释当前局势为什么变成这样：

- 异常最早如何改变世界。
- 关键战争、灾变、改革、背叛或迁徙留下了什么后果。
- 当前势力格局、禁忌、偏见或边界从哪里来。

### 8. 从历史推出势力

势力是历史和资源格局的行动者：

- 谁掌握资源或解释权。
- 谁维护制度，谁想推翻制度，谁只想活下去。
- 势力之间的联盟、敌意、交易和误判是什么。
- 它们对主角、关键人物和开局地点有什么压力。

### 9. 从势力推出普通生活

最后把宏观设定落到普通人每天如何生活：

- 普通人如何工作、学习、交易、旅行、求医、娱乐、信仰和避险。
- 异常规则如何进入日常语言、习俗、禁忌和常识。
- 主角的出身与日常困境如何体现这个世界。

## 够用阈值

世界书初始化结束前，至少能回答这些问题：

- 世界观基础：故事发生在什么样的世界，普通人如何理解这个世界。
- 物理因素：世界怎样运转，异常规则、力量体系、资源和限制是什么。
- 社会人文：社会怎样运转，制度、阶层、矛盾和普通生活是什么。
- 历史与当前局势：当前格局为什么形成，最近的压力点在哪里。
- 势力：谁掌握资源或规则，谁会帮助、利用或阻碍主角。
- 主角与关键人物：主角性格、外貌、能力或机制、初始状态、限制、关键关系、开局会遇到谁。
- 地图/位置：开局地点、邻近地点、危险边界或可探索方向。
- 冲突来源：世界规则如何把压力施加到主角身上，怎样推动开局剧情。

达成这些即可进入后续流程；不要为了完整性继续扩百科。

## 写入 lorebook

把稳定设定写入内容节点，可变当前状态写入 `state.md`。先更新已有节点，缺失时再创建新节点。

常用节点：

- `lorebook/world/rule/.../`：力量体系、制度、异常规则。
- `lorebook/system/.../`：可运行机制、玩法模块、状态规则。
- `lorebook/faction/.../`：组织、国家、家族、阵营。
- `lorebook/character/.../`：主角与开局关键人物。
- `lorebook/location/.../`：开局地点、邻近地点、关键地图节点。
- `lorebook/item/.../`：真正会影响剧情的物品或资源。
- `lorebook/note/.../`：总览、历史脉络、普通生活、待定但需要集中记录的设定。

写入时遵守：

- `summary` 保持短而可复用。
- 正文说明设定如何影响角色、社会和剧情。
- 用 `refs` 建立角色、地点、势力、规则之间的必要关系。
- 势力、地点、角色、物品的当前行动或当前处境写入对应 `state.md`，不要混进百科正文。
- `tags` 只写有意义、易理解、可复用的中文短标签；不要为了填字段随意设置。
- 不确定但重要的设定写入 `PROJECT-STATUS.md` 的 Pending Questions，或写成 `status: pending` 的 note，不要伪装成已确认事实。

## 主角、地点与力量体系

世界书初始化不仅写宏观规则，也要补齐开局会用到的具体对象：

- 主角设定：性格、外貌、能力或机制、资源、限制、初始处境、当前目标，以及主角和世界规则的关系。
- 人物创建：开局关键人物、重要关系和立场。
- 地点设定：开局地点、资源地点、危险边界、势力交界处。
- 力量体系：能力来源、成长路径、代价、限制、常见误解。
- 历史与当前局势：只写能解释当前冲突的历史，不写无关年表。
- 势力创建：势力目标、资源、手段、内部矛盾、对主角的压力。

这些对象可以随着推导链逐步出现，不需要先列全清单。

默认先做到“足够开篇使用”。如果用户想继续扩展，再沿历史、势力、地点、资源、普通生活等方向深入，并把扩展结果拆入对应内容节点。

## 后续衔接

世界书骨架完成后，如果用户要开始追踪世界状态、推进剧情时间线（谁在哪、何时发生了什么、当前状态如何），进入 `novel-workflow-world-engine-init` 初始化 World Engine：把需要追踪的角色、地点、势力登记为 subject，把 lorebook 里的力量体系、阶层、货币等设定映射成对应 schema 属性，并录入开局状态。

如果用户只想继续扩展静态设定，留在本流程沿历史、势力、地点、资源、普通生活等方向深入即可。

## 完成标准

- 已能清楚回答“这是个什么样的世界，和现实世界有什么不同”。
- 已能解释“世界怎样运转，物理规则、力量体系和资源逻辑是什么”。
- 已能解释“社会怎样运转，社会矛盾、势力、历史和普通生活是什么”。
- 世界基础、力量体系、关键势力、主角/关键人物、开局地点已经够开篇使用。
- 主角的性格、外貌、能力或机制、初始处境和限制已经能支撑开局写作。
- 世界规则能解释主角的处境、能力或机制，而不是和主角设定脱节。
- 关键设定已经进入 lorebook 真相源。
- 可变状态没有混进 `index.md`，需要时写入 `state.md`。
- 重要未定项被显式保留。
- 下一步可以直接进入 `novel-workflow-world-engine-init` 初始化 World Engine 追踪世界状态。
