# novel-technique-commercial-rhythm：商业网文节奏

这是技法 skill，不是完整写作流程。需要正式落库或写章时，回到对应 `novel-workflow-*` skill。

## 检查维度

- 主角当前目标是否清楚。
- 阻力是否具体，是否逼迫主角行动。
- 读者期待是否被建立、延迟、兑现。
- 低谷是否服务后续反击，而不是单纯压抑。
- 爽点是否有代价、铺垫和后果。
- 章末是否留下问题、危险、选择或反转。

## 使用方式

可以用于：

- `novel-workflow-07-opening-plot-design`。
- Plot 检查。
- 章节写后复查。
- 润色修订时调整节奏。

不要用它替代 lorebook、emulation tick、Plot System 或 writer。
