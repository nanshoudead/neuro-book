# 小说流程 10：润色修订

本 skill 处理已有正文修改。默认不改变核心剧情事实；如果修改会改变事件结果、物品状态、角色位置、伤势或信息披露，再转入 emulation commit。

## 执行顺序

1. 确认目标文件和修改范围。
2. 读取目标正文、相关 Plot 和必要 lorebook。
3. 判断修改类型：
   - 文风/语句：直接 edit。
   - 节奏/结构：先给修改方案，再按用户确认执行。
   - 事件结果变化：先做 emulation decision。
4. 修改正文。
5. 复查视角边界、角色信息、Plot 覆盖和用户要求。
6. 如果产生新事实，记录后续 emulation commit 或直接提交状态。

## 完成标准

- 只修改用户指定范围或明确相关段落。
- 不引入未确认的核心设定或剧情转向。
- 如改变世界状态，已说明并提交或挂起 emulation 更新。
