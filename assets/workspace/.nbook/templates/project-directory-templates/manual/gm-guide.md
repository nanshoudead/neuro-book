# 主持人手册

本文件是当前 Project 的人类主持人手册。它应该教新手主持人如何开第一场、解释规则、回答玩家问题，并控制剧透边界。

## 主持人身份

主持人描述世界、扮演 NPC、裁定后果，并让游戏继续推进。主持人应该公平、生动、能回应玩家，但不应为了强行达成某个结果而抹掉世界因果。

## 第一场游戏

描述推荐开局：

- 玩家从哪里开始。
- 哪些人物在场。
- 眼前的直接问题是什么。
- 玩家第一步可以做什么。
- 哪些信息玩家可见，哪些信息暂时隐藏。

开局前先确认玩家使用预设角色，还是按 `manual/player-guide/character-creation.md` 创建角色。使用预设角色时，可以跳过创建阶段说明，直接进入开场。

## 开场

开场发生在第一次行动之前。它应该让玩家立刻进入角色处境，而不是先听完整设定讲解。

好的开场通常包含：

- 身体感受。
- 周围环境。
- 在场人物。
- 直接压力。
- 异常线索。
- 第一个选择点。

## 场景节奏

跑团时使用 tick 节奏：

1. 描述玩家能感知的处境。
2. 询问玩家做什么。
3. 裁定行动。
4. 展示后果。
5. 玩家卡住时给出新的可选方向。

一个 tick 表示世界已经对一次行动完成回应。它可以很短，也可以覆盖一次复杂行动；关键是行动后必须出现新的局面或选择点。

## NPC

对重要 NPC，记录他们想要什么、害怕什么、知道什么。NPC 不应泄露自己没有理由泄露的秘密，也不应无理由违背自身利益。

## 桌面风格

记录默认风格和桌规预期：

- 难度和失败代价。
- 主持人给多少提示。
- 本桌偏好沉浸、共创、战斗、悬疑、恋爱、喜剧还是严肃剧情。
- 剧透容忍度。

## 剧透与提示

先给线索，再给答案。玩家请求帮助时，优先使用观察、直觉、传闻或 NPC 反应，而不是直接讲隐藏真相。如果玩家要求剧透，先确认剧透范围。

## 记录边界

记录已确认事件、角色变化和桌规偏好。不要把玩笑、假设讨论或桌外脑洞自动变成战役事实。
