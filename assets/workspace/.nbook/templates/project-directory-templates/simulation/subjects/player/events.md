# Player Events

本文件记录玩家角色在故事内经历、观察、听说或被告知的重要事件流水。它解释“玩家角色怎么知道某件事 / 状态怎么变化”，不写上帝视角真相。

## 起始

### 抵达起始场景

玩家角色刚抵达起始场景，附近有一名可以交流的 NPC。更具体的地点、时间和缘由以 GM 开场说明为准。

### 示例：获得物品

如果玩家角色在故事内获得三瓶普通血药，这里记录“从谁那里获得、何时获得、玩家角色知道它通常用于恢复伤势”。稳定认知整理到 `knowledge.md`，当前持有数量由 GM 写入 `state.md`。

如果玩家角色获得一瓶被下毒但自己不知道的血药，这里只记录“获得一瓶看起来普通的血药”。有毒真相写入 `simulation/entities/{entity-id}/state.md`，除非角色在故事内发现，否则不要写进本文件。

## 更新规则

- 只记录故事内已经发生、玩家角色已经经历或已经被告知的事件。
- 不替用户写未选择的行动、长期目标或内心决定。
- 不写 GM 隐藏裁决、其他角色私密意图或完整 lorebook 真相。
- 如果事件带来知识变化，把稳定认知整理到 `knowledge.md`；如果事件带来当前状态变化，由 GM 裁决后写入 `state.md`。
