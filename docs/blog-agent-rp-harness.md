# 论 Agent 与创意写作及角色扮演

创意写作及角色扮演本质上是一个大型的 RAG 工程。
即 indexing -> retrieval -> augment -> generation

索引：录入世界书（lorebook），人物，世界观，规则，人物关系等设定，向量化，建立graph
检索：根据当前剧情、故事、用户需要。agent 主动检索相关信息。这是为了解决 LLM 上下文长度过短的问题
增强：检索后的信息通当前用户制定发送给 LLM
生成：LLM 生成优化后的结果

对于 silly tavern。

索引：数据库，总结（通常依靠插件）
增强：预设，或者前端卡

生成：通常只有一次 LLM 调用（1 turn）

## indexing（建立索引）

写作和角色扮演是一个比较宽松的能力，天生适合 AI 帮忙

对于写作和角色扮演，他们都有一个最大的问题是：剧情设计和上下文管理（长期记忆）

这个问题细化到写作和角色扮演这两个领域变成了小问题

### 写作

分两种情况：全自动与半自动

#### 全自动：

由 AI 完全控制创作过程，用户给出一个题材或灵感，Agent 完全自主创作。这一点在这里不进行细化讨论（模型性能不行）

#### 半自动：通常由人类设计剧情，AI动笔、润色

人类解决了 AI 最大的短板：创意剧情设计。

此时最大的问题变成了上下文管理

### 角色扮演

对于角色扮演，通常是作者发布 “角色卡”，用户加载 “角色卡” 进行游玩。

通常有比较完善的世界观和事件。剧情问题影响不大。重要的还是上下文管理

同时用户可能在游玩的过程中自己新建设定，所以设定集（世界书）是一个动态变化的过程

### NeuroBook 对于写作、角色扮演提供了什么解决方法？

1. 为了防止作者和 LLM 在创作的时候跑偏，NeuroBook 制定了一套标准的目录规范
2. NeuroBook 划分了 lorebook 和 simulation 两个目录

lorebook 目录：这是上帝视角的设定，全知。同时还设定了原型
