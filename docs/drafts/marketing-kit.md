## 2. 电梯陈述

30 秒版：

> AI 能写好一段文字，但写不好一部长篇：写到几十万字，设定漂移、时间线混乱、伏笔埋了就忘，而且整章丢给模型，回来的文字一股 AI 味。NeuroBook 把长篇创作当成 IDE 问题来解决：世界状态由 World Engine 用时间线管理，不靠模型记忆；剧情和伏笔在剧情工坊里记账——每个伏笔都是对读者的承诺，写到那一章自动提醒兑现；写作由 leader、writer、检索等多个 Agent 分工完成；成稿再过一遍 340 条规则的 llmlint，把 AI 味筛出来。整部作品是你本地的 Markdown 文件和 SQLite，随时带走。

90 秒版（在 30 秒版基础上追加）：

> 具体来说，World Engine 用「时间线 + 切面」做事件溯源：每个重要时间点记录一次状态变更，任意时刻的世界状态都能推算出来——角色三个月前受的伤、王国十年前的国库，随时可查。世界结构用 Zod schema 自己定义，历法可以完全架空。剧情侧有剧情工坊：承载树管章节结构、因果树管剧情因果，承诺账本跟踪每个伏笔的埋设、推进和兑现，写到目标章自动出现在写作指令里。写作侧，默认主链是灵感探索 → 世界书初始化 → World Engine 建档 → 剧情规划 → 章节写作 → 写后回补，每个 Agent 有独立的工具权限：leader 可以改世界状态，writer 只能读。部署上 Windows 解压即用、服务器一行命令拉 Docker 镜像，模型 API Key 自己配，数据不出自己机器。

## 4. 四大核心能力（大杀器页素材，均无竞品对应物）

### 支柱一：World Engine —— 不吃书的世界状态引擎

副标题：任意时刻的世界状态，随时可查、永不漂移。

要点（slide bullets）：

- 「时间线 + 切面」事件溯源：每个重要时间点记录状态变更，任意时刻状态由之前切面推算得出。
- 补设定 = 在合适的时间点插一个切面；倒叙、回忆天然支持。
- 世界结构自己定义（Zod schema）：人物、门派、王国、大陆都可以是有状态的 subject。
- 历法自定义：现实公历、简化纪年、完全架空的历法都支持，公元前也能算。
- Agent 沙箱读写分权：leader 可写、writer 只读，写作时不会误改世界。
- 剧情 Scene 锚定世界时间轴、地点和出场角色，剧情规划与世界状态互相咬合。
- 每一次状态变更都是一条可审计的记录，引擎还会主动指出矛盾（如引用了不存在的主体），带解释、分级别。

讲稿金句：

> 长篇最大的敌人不是写不出来，是写着写着世界散架了。

> 谁在什么时候变成了什么样，引擎都记录在案；写出矛盾，它会主动指出来。

> 设定卡片记录「他是谁」，World Engine 记录「他在每一刻是谁」。（面对"和设定库有什么区别"时用）

方法论血统（name-drop 用）：软件工程的**事件溯源**（Event Sourcing，Martin Fowler）× 创作圈的**设定圣经**（Story Bible）——第一个把"设定圣经"做成可计算引擎的工具。

演示建议：Workbench 里选一个 slice 点「查询此时状态」，展示同一个角色在两个时间点的不同状态。

### 支柱二：Plot Workbench 剧情工坊 —— 两棵树 + 承诺系统

副标题：结构归结构，因果归因果，伏笔有账本。

要点（slide bullets）：

- **两棵树**：承载树（卷 → 章，管故事在哪讲）+ 因果树（阶段 → 剧情线 → 场景，管故事为什么发生）。倒叙、插叙、多线随便排，因果链永远清晰——叙事学讲了一百年的「故事与叙述分离」，第一次成为可操作的数据结构。
- **承诺系统（Promise）**：每个伏笔都是对读者的一个承诺——埋下（plant）→ 推进（advance）→ 兑现（payoff）。节拍挂在场景上，剧情写到哪、账本自动跟到哪；写到目标章，该还的承诺自动进入 writer 指令。四类承诺：伏笔回收、预言、母题、镜像。
- **创作决策记录（Decision）**：「为什么让主角黑化」当场记下，风险必填，推翻的决定也留痕——三个月后的你不用考古。
- **章节信息控制（ChapterBrief）**：读者知道什么、主角知道什么、必须隐瞒什么、只能暗示什么——悬念不再靠感觉，靠字段。

讲稿金句：

> 每个伏笔都是你对读者的承诺——NeuroBook 替你记账，到期提醒兑现。

> 三个月前你为什么让主角黑化？你忘了，Decision 记得。

> 别人的伏笔表要你自己记得去看；承诺账本写到那一章，自动催账。

方法论血统（name-drop 用）：**布兰登·桑德森的 Promise / Progress / Payoff** 三法则 + **契诃夫之枪** + **希区柯克的悬念理论**（信息控制字段）+ 软件工程的 **ADR 决策记录** 与 **技术债追踪**（Ward Cunningham——"债"这个隐喻的原始出处）。

演示建议：建一个承诺 → 在两个 Scene 上打 plant / payoff 节拍 → 展示目标章 brief 自动带出「本章承诺任务」。

### 支柱三：多 Agent 写作工作室 —— 好马配好鞍

副标题：检索、规划、写作、审校，各归其位。

要点：

- 默认写作主链：灵感探索 → 项目与世界书初始化 → World Engine 建档 → 剧情规划与状态推进 → 章节写作 → 写后回补。
- leader 负责剧情规划与调度，writer 专职正文，retrieval / researcher 查设定查资料——数值不乱编（引擎账上有）、资料不瞎猜（researcher 去查）。
- 每个 Agent 有独立的工具白名单、上下文模板和压缩策略。
- **三模式**：讨论模式只出主意不动稿，计划模式先给完整方案、批准才执行，每次模式切换都要你点头；只读模式下的写操作自动挂起审批，批准后真实执行。
- **文件历史**：Agent 动过的每个字有案可查——单文件历史误删可找回，收件箱逐条看 diff、逐条接受或撤销，重开老会话自动知道"你不在时发生了什么"。
- 编辑器内 Inline AI：选中即改、流式预览 diff、不打断主编辑流。

讲稿金句：

> AI 已经是一匹好马，但大多数人还在裸缰骑——NeuroAgentHarness，Harness 本义就是马具：好马，配好鞍。

> 你不会让一个员工同时当编辑、资料员和写手，为什么让一个模型这么干？

> 事前有审批（三模式），事后有追溯（文件历史）——你的稿子永远在你手里。

> 你面对的不是一个冰冷的陌生工具，而是一个自带「器灵」的工作室：配好 API Key，剩下的路器灵带你走。（仅网文渠道口播）

方法论血统（name-drop 用）：Code Review 与 Terraform 的 **plan / apply**（三模式）、**最小权限原则**（工具白名单）、IDE 的 **Local History**（文件历史）。

演示建议：discuss 模式聊剧情 → 切 plan 模式出方案（审批切换）→ normal 模式让 leader 调 writer 产出正文 → 收件箱里看这一轮改了哪些文件。

### 支柱四：llmlint —— 给文字做 lint

副标题：写完的最后一步，去掉 AI 味。

要点：

- 340 条规则（311 条默认启用），覆盖填充词、机械过渡、公式化设问、二元对比、空泛总结、节奏单调等典型 AI 痕迹。
- 双层规则：静态正则秒级扫全稿，LLM 规则做语境判断。
- 类 eslint 体验：多文件 / glob、Markdown 遮罩、JSON / stylish 输出、机械问题自动修复。
- 既是 NeuroBook 内的润色 Skill，也是独立 CLI 工具：github.com/notnotype/llmlint。

讲稿金句：

> 代码有 linter 三十年了，文字现在也有了。

方法论血统（name-drop 用）：**lint**（贝尔实验室，1978）× 奥威尔《政治与英语》的写作六规、《风格的要素》（Strunk & White）——把风格圣经变成可执行的规则。

演示建议：现场对一段典型 AI 生成文字跑 `llmlint check`，展示命中报告和修复前后对比。

## 7. 竞争格局与应对口径（不点名版）

**开场事实（不需要教育市场）**：AI 已经参与了绝大多数作者的创作过程——但形态是网页聊天框：没有记忆、没有工具、通用提示词。用马做比：**模型是一匹一年比一年好的马，可大多数人还没给它配鞍。** NeuroBook 的位置就是那副鞍（NeuroAgentHarness，Harness 本义即马具）：长期记忆（世界引擎 + 承诺账本 + 文件历史）、写作特调的提示词与方法论（profile / skill）、剧情设计工具（两个工作台）。

对外文案与 PPT **不点名任何具体竞品**，只按「工具类型」划分。每类给一句区隔口径，可直接当讲稿：

| 竞品类型 | 他们解决什么 | 一句话区隔口径 |
| --- | --- | --- |
| AI 聊天框 | 生成一段文字 | 模型负责生成，NeuroBook 负责让几十万字的工程不散架。 |
| 角色扮演工具（酒馆类） | 角色对话体验 | worldbook 是静态词条，World Engine 是随时间演化、可推算的世界。 |
| 静态设定库型工具（codex / 设定卡片 / 设定树） | 结构化管理设定 | 卡片记录「他是谁」，World Engine 记录「他在每一刻是谁」——设定卡是死档案，世界是活的。 |

## 7.5 产品立场页（价值观 slide，放在四大核心之后）

标题：**人类主导，Agent 执行。**

- 我们不做「睡一觉起来多了五章」：当前模型无人值守的产出质量守不住，挂机量产出来的是要重写的字数，不是作品。
- NeuroBook 的分工是：作者掌握每一个创作决定，Agent 负责检索、推演、执行和记账。
- 立场有机制背书，不是口号：**三模式**（讨论 / 计划 / 正常）让"要不要开始动手"也归你审批；**文件历史**让每次动手都可追溯、可撤销——事前有审批，事后有追溯。
- 检索是主动的：RAG 不是在背后无脑注入相似片段，而是作为工具交给 Agent，让它带着意图自己判断何时该查、查什么（World Engine 经 CodeAct 已带 RAG 工具；lorebook 文件当前用 rg 等文件工具精确检索）。带意图的检索比被动相似度召回更准。

讲稿金句：

> 器灵替你跑流程，但每一个创作决定都是你按的。

> AI 是器灵，不是代笔。

## 7.6 定位页：我们做基础设施，功能由社区长出来（价值观 slide）

标题：**NeuroBook 是底座，不是全家桶。**

- 别的工具比谁的功能清单更长：关系图、AI 封面、拆书、伏笔表……做成一个个焊死的按钮。
- NeuroBook 反过来：把 World Engine、多 Agent、文件化 workspace、Skill 机制做成**可编程的地基**，具体功能作为 Skill 在上面生长。
- 好处对作者：你要的功能不必等官方排期——它可以是一个 Skill，你自己写、或从社区拿。
- 好处对生态：关系图、封面生成、txt 拆书迁移这类需求，都能由社区 Skill「长出来」，而不是挤在一个团队的路线图里。

讲稿金句：

> 与其给你一百个焊死的按钮，不如给你长出一百个按钮的土壤。

对外纪律（内部备注，不上 slide）：存量迁移、关系图 / 封面这两类，统一归到「Skill 生态会长出来」的框里，**不做功能承诺、不进主推清单**；被追问才用本页口径接住，绝不主动演示未完成能力。（伏笔追踪已随承诺系统解禁，从本清单移除。）

## 8. 目标用户画像（「为谁而做」页）

1. **长篇网文作者**：几十万字以上、多卷本，苦于设定表和时间线维护。核心钩子：World Engine。
2. **AI 辅助写作重度用户**：已经在用 AI 写作但被 AI 味和上下文限制折磨。核心钩子：llmlint + 多 Agent。
3. **SillyTavern / RP 玩家**：手里有大量角色卡和 worldbook 资产，想升级到有状态的世界。核心钩子：角色卡迁移 +（未来）RP 模式。口径注意：RP 模式表述为「重新设计中」。
4. **自部署爱好者**：要求数据本地、模型自选。核心钩子：本地文件 + 一行部署。
5. **没受过专业训练的写作爱好者**（定位升级后的新增人群）：心里有一部长篇但不知道怎么开始、怎么撑到写完。核心钩子：方法论内嵌——不用读过写作理论，工具替你带着走（信息控制字段替你问"读者知道吗"，承诺账本替你记"坑填了吗"）。

## 9. 演示脚本（Live Demo 流程，约 10 分钟）

1. **开场（30s）**：打开 Bookshelf，展示一个已有项目的 workspace 文件树——「这就是一部作品的全部，都是文件」。
2. **World Engine（2.5min）**：打开 Workbench → 看 timeline → 选一个历史 slice 查询「此时状态」→ 现场写一个切面（如「主角受伤」）→ 再查询状态变化。强调：这些不是笔记，是可推算的状态。
3. **剧情工坊（2min）**：展示两棵树（承载树 / 因果树）→ 建一个承诺、在两个 Scene 上打埋下 / 兑现节拍 → 打开目标章 brief，展示「本章承诺任务」自动出现。强调：伏笔不靠你记得，是系统管道。
4. **写作主链 + 三模式（3min）**：discuss 模式和 leader 聊下一章思路（只出主意不动稿）→ 审批切到 plan 模式看完整方案 → 批准进 normal 模式，leader 读世界状态、更新 Scene、调用 writer 产出正文。强调：每一步升级都要你点头。
5. **llmlint（1.5min）**：对刚生成的正文跑润色 Skill / CLI，展示命中规则和修复。
6. **收尾（30s）**：回到文件树——「刚才所有产出都落在这些本地文件里，打包就能带走」。

Demo 风险预案：现场模型调用可能慢或失败，提前录一段备用视频；World Engine / 剧情工坊查询是纯本地计算，网络差时把演示重心放在步骤 2-3。

## 10. 技术架构页（技术受众场合选用）

- 技术栈：Nuxt 4（Vue 3，实测 4.3.1）+ Bun runtime + SQLite（App 级 + Project 级双库）+ Prisma 7。
- Agent 底座：NeuroAgentHarness，基于 Pi 框架扩展（multi-provider、tool calling、append-only session tree），支持 Multi-Agent、HITL、运行时 Profile / Tool Catalog、上下文压缩、Runtime Hooks。

## 11. 反对意见应对（Q&A 备稿）

- **「为什么用 AGPL 开源？」**——任何人都可以使用、研究、修改、分发和商业部署；分发修改版或用修改版向网络用户提供服务时，需要按 AGPLv3 提供对应源代码。**作者用它创作的原创作品不因使用软件而自动适用 AGPL，商业出版不受影响**。
- **「我的稿子会不会被拿去训练？」**——数据全在本地文件和本地数据库，模型调用走你自己配置的 Provider，项目方不经手内容。
- **「模型费用谁出？」**——自带 API Key，多 Provider 可选。token 统计比市面绝大多数 Agent 工具透明：输入、输出、缓存创建、缓存命中分项计量，并直接换算成美元 / 人民币——你能确切知道「写这一章花了多少钱」。
- **「和直接用 Claude/GPT 写有什么区别？」**——回到痛点页：模型负责生成，NeuroBook 负责让几十万字的工程不散架。
- **「为什么不能挂机自动写几十章？」**——立场问题，不是能力问题：当前模型无人值守的产出质量守不住，挂出来的是要重写的字数。NeuroBook 的分工是人类主导创作决定，Agent 负责检索、推演和执行。
- **「装个编码 Agent 的写作插件不就行了？」**——三点：①那是终端工具，普通作者要先学会用命令行；NeuroBook 是 Web 界面，打开就是写作台。②那类插件锁定单一厂商订阅；NeuroBook 多 Provider 自由。③编码 Agent 的系统提示词和工具面是为写代码设计的，等于在编码工具里借道写小说；NeuroBook 的上下文工程从第一行就为叙事而生。
- **「和设定卡片 / 设定树类工具有什么区别？」**——卡片记录「他是谁」，World Engine 记录「他在每一刻是谁」。设定卡是死档案，改了就把旧状态抹掉；World Engine 的每次变更都是带时间戳的可审计记录，任意时刻状态可回放，写出矛盾（比如引用了不存在的人物）引擎会主动报出来。
- **「伏笔追踪有吗？」**——有，而且是一等公民：承诺系统。每个伏笔是一笔对读者的承诺，埋下 / 推进 / 兑现全程记账；节拍挂在场景上随剧情自动更新，写到目标章自动进入 writer 指令。和手工伏笔表的区别：账本长在剧情结构上，不用你记得去看。
- **「几百万字的存量作品怎么迁进来？」**——当前通过 Agent + 拆书类 Skill 处理；一键式 txt 导入在 Skill 生态路线图上。（诚实口径：还不是一键，不要承诺。）
- **「有没有人物关系图 / AI 封面这些？」**——不内置全家桶，走 Skill 生态扩展；当前主战场是「让长篇不写崩」这条主链。
- **「RP 功能呢？」**——RP / 世界模拟能力在底层保留，入口正按写作模式的体验标准重新设计；当前版本主打写作模式。
- **「成熟度如何？」**——v0.5 快速迭代阶段，写作模式主链已跑通端到端验证；适合尝鲜共建，不承诺 API 稳定。

### 14.1 Promise 规划层：读者债务账本（Task 93 + 前置 Task 97）

**状态（2026-07-08）**：设计定稿（三轮修订收口），实施进行中——Slice 1-2（数据层 + 维护/读取工具）已实现并测试，Slice 3（brief 消费）遇工具门控架构问题待定夺；工具面按 Task 97 新形态落地（save_* 合并 / 命名统一 / 分层绑定）。

**解禁条件**：Slice 3 收口 + `bun test server/plot` 全绿 + 真实项目实测通过（建 Promise → 打 plant/advance/payoff beats → 派生状态正确 → 目标章 brief 自动编出「本章 Promise 任务」段与 open Decision 警告段）。

**一句话用户价值**：伏笔不再靠作者的记性和便签——每个伏笔都是你欠读者的一笔债，系统替你记账、跟踪、到期催收，写到那一章自动把该还的债塞进写作指令。

**机制细节（文案素材，全部来自 Task 93 定稿设计）**：

- **心智模型**：Promise = 对读者的承诺 / 债务。生命周期 plant（埋下）→ advance（推进）→ payoff（兑现）。统一模型、字段驱动、**无 kind 类型分裂**——一个概念讲到底，作者不用学分类学。
- **伏笔四分类 tags**（比竞品单一"伏笔"概念深一层，编剧理论工程化）：
  - `setup_payoff`：经典埋点回收（契诃夫之枪）。
  - `prophecy`：预言 / 誓言 / 约定，读者明知要应验、等的是怎么应验。
  - `motif`：母题 / 意象的反复出现。
  - `mirror`：镜像 / 对照结构（开头结尾呼应、双线互文）。
- **beat 挂在 Scene 上，计划 / 事实 / archived 三态派生**：伏笔的每一步（beat）挂到具体 Scene；Scene 还没写是「计划」，写完自动变「事实」，Scene 归档 beat 跟着归档。**这是与竞品伏笔注册表的本质区别：竞品是一张要你自己记得去看的孤立表格，NeuroBook 的账本长在剧情结构上，剧情走它跟着走。**
- **fulfilled 可回退**：兑现状态是派生的——只有当没有剩余 payoff 计划时才算还清；后续又加了 payoff，状态自动回退。账本永远反映真实债务，不会"标完就死"。
- **Decision：ADR 式创作决策记录**：记下「为什么当时决定这么写」，**risk 字段必填**（逼你想清楚这个决定的代价），状态含 dropped（推翻的决定也留痕）。三个月后想改设定，先看看当时为什么这么定。
- **cadence 节奏字段**：提示性节奏标注，不是硬约束——不锁死创作。
- **brief 自动消费**：目标章的 writer brief 编译时自动插入「本章 Promise 任务」段（该埋的、该推进的、该还的）+ 未拍板 Decision 警告段。**伏笔进入写作上下文不靠作者提醒，是系统管道**。默认详细 brief、默认假设 writer 忠实执行，写后核对交给后续 critic 任务。
- **章序坐标系**：deadline 等用章节顺序表达，贴合连载作者的真实心智（"第 50 章前要收"）。
- **工程严谨性**（技术受众可讲）：outcomeType 含 `passive`、`null` 不承载语义（全系统 D29 显式语义原则）——没有"没填就是默认 XX"的暗坑。

**竞品对照（内部参考）**：同类工具的伏笔功能 = 孤立注册表 + 手工状态机（开启/悬置/消费）；无一做到「挂在剧情结构上自动派生 + 自动进入写作上下文」。解禁后 §7 对比表可新增「伏笔登记 / 推进 / 兑现」行并打 ✅（注明"自动进 brief"）。

**金句候选**：

> 每个伏笔都是你欠读者的一笔债——NeuroBook 替你记账，到期催收。

> 三个月前你为什么让主角黑化？你忘了，Decision 记得。

> 竞品的伏笔表要你自己记得去看；NeuroBook 的账本写到那一章，自动催账。

### 14.2 三模式 Agent：normal / discuss / plan（Task 90）

**状态**：已实现（typecheck / 测试全绿），待浏览器验收。**解禁条件：浏览器验收通过。**

**一句话用户价值**：Agent 不会自作主张动你的稿子——想聊剧情用讨论模式（只出主意不动手），想干大活用计划模式（先给完整方案、你批准才执行），每一次模式升级都要你点头。

**机制细节**：

- 三模式：normal（正常读写）/ discuss（讨论，只读）/ plan（规划，只读 + 产出计划文件）。
- `switch_mode` 工具：Agent 想切换模式必须发起审批，用户批准才生效——**连"要不要开始动手"这个决定都在你手里**。
- 只读模式下的写操作自动挂起为审批请求，**批准后真实执行**（不是假拒绝再重试）；plan 目录豁免（计划文件本身可写）。
- 前端 Shift+Tab 循环切换、按模式区分 placeholder；reminder 按 模式 × 阶段 矩阵渲染，模型不会"忘了自己在什么模式"。

**营销用途**：§7.5「人类主导，Agent 执行」立场页的**机制证据**——立场从价值观宣言变成可演示的产品能力，这是它最大的宣传价值。

**金句候选**：

> 讨论模式聊剧情，计划模式看方案，正常模式才动笔——每一步升级都要你点头。

**解禁后动作**：§7.5 加「机制支撑」三行；§9 demo 脚本在写作主链前插 30 秒 discuss → plan → normal 切换演示；§5 次级卖点可加一条。

### 14.3 文件历史 / 操作日志（Task 91 spike + Task 95 集成）

**状态**：独立模块 `nb-history` spike 已实现并通过验收（31 测试全绿多轮 0 fail、tsc 全绿；性能实测：单次 30KB 写入 8.5–12ms，5000 条积压查询 18–35ms，单文件 100 版 × 30KB 存储开销仅 ~5%——内容寻址去重生效）；**未集成进 NeuroBook**（Task 95 blocked on Task 94 生命周期改造）。**解禁条件：集成进产品且 UI 可用。**

**一句话用户价值**：Agent 动过的每一个字都有案可查、有据可回——误删的文件能找回，改动逐条看 diff、逐条接受或撤销。

**机制细节（文案素材）**：

- IDEA Local History 式单文件历史：每次变更留版本，误删也能找回。
- 收件箱：人改了什么、Agent 改了什么，互相看得见；逐条 accept / revert（Monaco diff）。
- 多会话互知：重开老会话，自动知道"你不在的时候发生了什么"——每会话游标，diff 基准数学精确（该会话最后见过的状态）。
- 记录所有主体（n 用户 × n 会话 × 外部操作）的文件操作，可回溯。
- 隐私红线兼卖点：history.sqlite 含正文，**永远被排除在可分享日志包之外**——你的历史只属于你。

**营销用途**：终结「AI 会不会乱改我稿子」的恐惧；与 14.2 三模式组成完整安全故事——**事前有审批（模式），事后有追溯（历史）**。

**金句候选**：

> Agent 动过的每一个字，都有案可查、有据可回。

**解禁后动作**：进 §5 次级卖点 + §7.5 立场页安全故事；自部署 / 技术渠道标题可用「本地文件历史」钩子。

## 15. 元叙事：两个领域方法论的系统移植（主叙事）

**主张（2026-07-08 升级为双血统版）**：NeuroBook 的每个核心功能都有双重血统——软件工程的成熟实践 × 创意写作的经典理论。竞品是单点借鉴，NeuroBook 是**成体系移植**，并且把理论做进工具，让没受过训练的人也能用专业方法写作（方法论民主化）。

**双血统对照表**（长文 / 技术分享核心素材，全部已并入产品叙事）：

| NeuroBook 功能 | 软件工程血统 | 创意写作血统 |
| --- | --- | --- |
| World Engine | 事件溯源（Event Sourcing，Martin Fowler） | 设定圣经（Story Bible）、托尔金式世界构建 |
| 承诺系统 | 技术债追踪（Ward Cunningham，"债"隐喻的原始出处） | 桑德森 Promise / Progress / Payoff 三法则、契诃夫之枪 |
| 章节信息控制 | 最小权限 / 信息隔离 | 希区柯克「桌下炸弹」悬念理论 |
| 两棵树 | 关注点分离 | 俄国形式主义 fabula / sjuzhet（故事与叙述分离） |
| Decision | ADR（Michael Nygard） | 金圣叹批《水浒》、脂砚斋批《红楼》的评点传统 |
| llmlint | lint（贝尔实验室，1978） | 奥威尔《政治与英语》、《风格的要素》 |
| 三模式 + 审批 | Code Review、Terraform plan / apply | 编辑部三审制 |
| 文件历史 | Local History / VCS | —— |
| lorebook（90% 不进正文） | —— | 海明威冰山理论 |

**名字弹药精选（用户拍板：只抛最权威、最有噱头的）**：

- **一军（主打，任何场合可抛）**：契诃夫之枪 · 希区柯克悬念理论 · 布兰登·桑德森三法则 · 海明威冰山理论 ｜ 事件溯源 · lint · ADR · 技术债（Ward Cunningham）。
- **二军（深度长文才抛，浅场合不堆砌）**：fabula / sjuzhet（什克洛夫斯基）· Save the Cat · 麦基《故事》· 金圣叹 / 脂砚斋评点 · 奥威尔六规。
- 纪律：一篇物料最多抛 3-4 个名字，**每个名字必须挂在一个具体功能上**（"信息控制字段=希区柯克炸弹理论的工程化"），禁止裸列名词充门面。

**主叙事句**：

> 软件工程用三十年学会了让千万行代码不崩，创意写作用一百年总结了怎么讲好故事——NeuroBook 把两边的学问，做成了同一个 IDE。

**方法论民主化（定位升级的核心论证）**：

- 逻辑链：桑德森、麦基、希区柯克的理论一直都在，但用它们的前提是先读完并内化 → NeuroBook 把理论做成字段、账本和工作台 → **不读理论也能用理论**，写作门槛坍塌。
- 类比弹药：Photoshop 把暗房做成滑块；Excel 把会计做成表格；NeuroBook 把叙事学做成工作台。
- 同时解决的实际问题：忘事（Decision + 承诺账本 + 文件历史 = 外置记忆）；专业失误 / 数据乱编（数值在引擎账上、资料由 researcher 去查——口径收着说："比裸聊天框少一个数量级的胡编空间"，不说"杜绝"）。

### 15.5 黄金圈叙事（Simon Sinek，演讲 / 官网首屏用）

- **Why**：每个人心里都有一部长篇，但绝大多数死在半路——不是死于没天赋，是死于没有工程。我们相信，讲完一个好故事不该以"天赋 + 十年训练"为门票。
- **How**：把两个领域百年的方法论——软件工程的事件溯源、lint、ADR，与创意写作的桑德森三法则、契诃夫之枪、希区柯克悬念——做成人和 AI 共用的同一套工具。
- **What**：一个创意写作 IDE：世界引擎、剧情工坊（两棵树 + 承诺系统）、llmlint、多 Agent 写作工作室，作品全部在你本地。
- 用法：演讲从 Why 讲起（愿景开场），发帖从痛点讲起（What 的反面）；两者共用 §3 / §4 素材。

**使用规则**：

- 适用渠道：LINUX DO / 知乎长文 / HN / 技术分享的开场框架；README 升级的候选方向。
- 网文渠道口播可用「器灵」拟人（见 §13 术语表），书面物料一律用直白表述；「天道 / 因果簿」已弃用。
- 此叙事与 §7.6「底座而非全家桶」互相支撑：双血统是"移植了什么"，底座是"以什么形态提供"。
