# Round 378 - Runbook Time Window Preflight

## 背景

Round 376 的真实浏览器验收 runbook 固定使用 `复兴纪元488年 1月15日 14:00:06` 到 `14:00:09` 作为验收 slice 时间。由于 World Engine 当前约束是同一 instant 只能有一个 slice，正式浏览器验收前需要确认这些时间点仍未被占用。

本轮只做只读检查，不写入 Project SQLite，不执行浏览器验收。

## 查询方式

通过 `WorldEngineFacade`：

- `parseTime(project, time)` 解析四个 runbook 时间。
- `listSlices(project, {from, to, withMutations: true, limit: 20})` 查询该时间窗口内已有 slice。

## 结果

时间解析结果：

- `复兴纪元488年 1月15日 14:00:06` -> `15148908006`
- `复兴纪元488年 1月15日 14:00:07` -> `15148908007`
- `复兴纪元488年 1月15日 14:00:08` -> `15148908008`
- `复兴纪元488年 1月15日 14:00:09` -> `15148908009`

窗口查询结果：

- `sliceCount: 0`
- 当前 `15148908006..15148908009` 没有已存在 slice。

## 结论

Round 376 runbook 当前可直接使用这四个时间点，不会撞同 instant 冲突。

正式浏览器验收如果不是立刻执行，执行前最好再做一次同样的只读检查；若期间已有新 slice 占用这些时间，应把 runbook 时间整体顺延，避免把验收失败误判为 UI 问题。

## 验证

本轮只运行只读 facade 查询，没有修改代码、没有修改真实 Project 数据、没有运行测试、没有执行浏览器验收。
