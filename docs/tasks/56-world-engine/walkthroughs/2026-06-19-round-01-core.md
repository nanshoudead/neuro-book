# Round 01 - Core World Engine

## Scope

本轮目标是先落后端核心闭环，不接前端、不接 `simulator.leader`、不做浏览器测试。

实现范围：

- Project SQLite 增加 `WorldSubject` / `WorldSlice` / `WorldMutation`。
- 新增 world-engine 后端模块，提供 subject 创建、slice 写入 / 编辑、显式 re-settle、reduce 查询。
- 支持从 Project Workspace `world-engine/schema.yaml` / `world-engine/calendar.yaml` 读取第一版配置。
- 增加核心单测，覆盖同 instant 冲突、`editSlice`、显式 re-settle、`queryState` 投影。

## Implementation Assumptions

- `editSlice` 第一版采用整块替换：`instant/title/summary/kind/mutations` 一次提交。
- `writeSlice` 遇到相同 instant 的已有 slice 直接报错，不做合并。
- `writeSlice` / `editSlice` 会结算当前 slice 自己的 `old`，但不自动修改后续 slice 的 `old`。
- `resettleTimeline(projectPath, { from, subjectIds })` 是显式重结算入口。
- `WorldSubject.name` 只是列表显示冗余，不是状态真相。
- `createSubject` 若目标 instant 已有 slice，作为系统初始化写入允许把 init mutations 追加到已有 slice；普通 `writeSlice` 不允许。
- Calendar 第一版只做 token 格式 parse / format，不允许用户写 JS 函数。
- Agent / API 层不接受 raw instant；工具层后续只收日历字符串。
- `getWorldSchema` 不返回 subject 实例；subject 实例后续由 `listWorldSubjects` 查询。

## Review Notes

- `old` 当前存储为 `{ exists: boolean, value?: JsonValue }` JSON，用于区分“原值为 null”和“原属性不存在”。这比直接存裸 old 值更啰嗦，但第一版更稳。
- `createSubject` 的 schema default 使用 `set` 写入，即便目标属性 kind 是 `list` / `collection`。这是初始化语义特例；普通 mutation 仍按 kind 限制 op。
- Calendar 第一版是固定进位 token 历法，不支持 gregorian、不规则月份、模糊时间和复杂自然语言。
- 测试在 Windows 下遇到 SQLite 句柄释放延迟，测试清理改为 afterAll 最佳努力；命令结束后已清理残留 `workspace/world-engine-test-*` 目录。
