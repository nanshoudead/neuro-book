# 2026-07-06 P2 fixes

## Scope

本轮修复 3 个 P2 问题：

1. Agent client variable patch 在切换主题时必须等待保存结果，不能提前 ack success。
2. 主题 JSON 导入必须校验每个变量值是可编辑的具体颜色。
3. World Engine preview 与普通 UI 硬编码色继续收口，`--we-*` preview 漂移要有单一映射源。

约束保持不变：不扩展 36 主题变量；8 套内置预设不改值；分类/内容色板不迁移；`--we-*` 别名层保留只改指向。

## Changes

- `app/components/novel-ide/agent/client-variables.ts`
  - `applyClientVariablePatch()` 改为 async。
  - setter 支持 `void | Promise<void> | boolean | Promise<boolean>`。
  - setter 返回 `false` 时抛错，让调用方 ack error。
- `app/components/novel-ide/agent/AgentChatSurface.vue`
  - client patch 处理改为 `await applyClientVariablePatch(...)`。
  - `ide.theme` setter 等待 `themeManager.setTheme(value)`，保存失败不再提前 ack success。
- `app/utils/theme/theme-io.ts`
  - 使用 `colord(value).isValid()` 校验导入 JSON 的每个 `vars` 值。
  - 接受 hex / rgb(a) / hsl(a) 等具体颜色；拒绝 `not-a-color`、`var(...)`、`color-mix(...)`、非法 hex。
- `app/styles/theme-vars.css`
  - 新增唯一 `.world-engine-workbench-theme` 映射源，集中定义 `--we-* -> v2.1` 主题变量。
- `app/components/novel-ide/world-engine/WorldEngineWorkbenchDialog.vue`
  - 删除 scoped style 中重复的 `--we-*` 映射。
- `app/pages/world-engine.workbench-preview.vue`
  - preview 根节点挂 `.world-engine-workbench-theme`。
  - 删除旧浅绿 `--we-*` 硬编码和 `--bg-main: var(--we-bg-canvas)` 等反向覆盖。
- 普通 UI 色继续收口：
  - TipTap / frontmatter / diff / dialog / model settings / plot tree / profile template 周边状态色和阴影改用主题变量。
  - Plot tree 主线连线改用 `--accent-main`，支线连线改用 `--text-muted`。
  - Reference chip 类型色只保留在 `app/styles/reference-chips.css`，组件不重复定义类型色。
- 文档同步：
  - `AGENTS.md`
  - `CLAUDE.md`
  - `app/utils/theme/README.md`
  - `reference/theme/system.md`
  - `PROJECT-STATUS.md`
  - `docs/tasks/89-theme-system-v2/README.md`

## Exceptions

- Reference chip 类型色是分类识别色，唯一外观源为 `app/styles/reference-chips.css`。
- Plot / Workspace tone 色板仍是分类色板，不迁移为主题变量。
- Profile template 节点类型 accent、Markdown 内容颜色选择器、JsonViewer / Monaco 语法色、备用 Markdown 内容主题属于内容或第三方编辑器色板。
- NotificationViewport 当前挂在 `.novel-ide-theme` 宿主外层，玻璃拟态和固定反色文本暂作为宿主外例外；若未来移入主题宿主，再按状态变量统一。
